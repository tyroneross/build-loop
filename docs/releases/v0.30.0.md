# v0.30.0 Release Notes

Covers the v0.29.x → v0.30.1 arc: post-push retrospective + backlog auto-iterate (v0.29.0), mandatory Phase 6 Learn + retro→Learn wiring (v0.30.0), the lazy-psycopg portability fix that unblocked full-suite collection (v0.30.0, peer 8750d2a), and the pytest-collection gate at Review-B (v0.30.1).

## Mandatory Phase 6 Learn (v0.30.0)

- Phase 6 Learn now runs after every build. The orchestrator emits a `## Learn` outcome line in the Review-G report on every run — no more "skip entirely" condition.
- Three outcome states cover every run:
  - **Accruing** — `runs[] < 3`. Detector + memory consolidation only; no Sonnet draft. Line: `Learn: accruing (N/3 runs)`.
  - **Deferred** — debug-only dispatch (`closeout: false`) or budget-exhausted at Phase 6 entry. Detector + consolidation run; writes `.build-loop/proposals/learn-deferred-<run-id>.md` marker; Sonnet draft + Opus signoff skipped. Line: `Learn: deferred — <reason>`.
  - **Full** — `runs[] >= 3` AND detector returned a pattern. Detector + Sonnet draft + Opus signoff + sample sweep. Line: `Learn: <N> patterns drafted` (or `Learn: 0 patterns above threshold (N runs scanned)` when nothing crossed).
- The expensive arm (Sonnet draft + Opus signoff) stays conditional. Net marginal cost over the prior gated path is one cheap Haiku state-scan per run.
- Migration no-op: `.build-loop/config.json.autoSelfImprove: false` is no longer honored. Old configs do not error — the orchestrator appends a one-line deprecation warning to `state.json.warnings[]` and proceeds as if the key were absent.
- Promotion to `active/` still requires explicit `/build-loop:promote-experiment <name>` (decision-3 safety boundary preserved). Auto-promote of unreviewed drafts never happens.

Source: `skills/build-loop/references/phase-6-learn.md`.

## Post-push retrospective (v0.29.0)

- New `retrospective-synthesizer` agent (Sonnet, non-gating) dispatched after the Phase 4 Report closing push. Writes a structured 9-section lessons-learned to `.build-loop/retrospectives/<YYYY-MM-DD>/<run-id>.md` plus a ≤5-line `<run-id>.summary.md` surfaced inline.
- Fire-and-continue contract: the orchestrator does not block on the agent's envelope before closing the run. Reads are local (transcript JSONL + `state.json` + intent + plan); writes are atomic via `os.replace`.
- Anything prompted ≥2× in the thread, or surfaced for the "what should be enforced" section, becomes an auto-drafted enforce-candidate at `.build-loop/proposals/enforce-from-retro/<run-id>-<NN>.md` — a candidate for human review, never silently promoted.
- Best-effort durable mirror to `build-loop-memory/projects/<slug>/retrospectives/` when the canonical memory store is reachable.

Source: `agents/retrospective-synthesizer.md`, `scripts/retrospective/`.

## Backlog auto-iterate (v0.29.0)

- Product-impacting work descoped during a build is triaged at capture time via `scripts/backlog/triage.py` + `scripts/backlog/assess.py` and written to `.build-loop/backlog/<id>.md` using `templates/backlog-item.md`.
- Frontmatter contract carries mandatory `repo` + `branch` segmentation keys plus `classify` (SAFE | RISKY | DECISION | PRODUCTION), `effort` (XS | S | M | L | XL), `status`, and `product_impacting` (bool).
- Phase 5 Iterate drains the backlog at end-of-run by default, alongside `issues/` and `ux-queue/`. The durable cross-run backlog of record lives at `build-loop-memory/projects/<slug>/backlog.md`.
- Segmentation is binding: a cross-repo item belongs in its own repo's scope. Never mix repos in one tracker. The freeform `OPEN-ITEMS.md` was retired 2026-05-31 for accreting cross-repo rows.

Source: `templates/backlog-item.md`, `scripts/backlog/`.

## Retro → Learn wiring (v0.30.0)

- `scripts/enforce_retro_signals.py` is the second signal source for the `recurring-pattern-detector` agent. It scans `.build-loop/proposals/enforce-from-retro/`, normalizes each candidate body, and counts recurrence across **distinct run-ids**.
- The detector emits `enforce_recurrence` patterns when the same normalized candidate signature appears across ≥ 2 distinct run-ids — delivering "anything prompted or needed repeatedly → enforce" *across* sessions, not just within one.
- Output envelope matches the detector's existing `patterns[]` shape so the agent splices the signal directly.

Source: `scripts/enforce_retro_signals.py`, `agents/recurring-pattern-detector.md`, `skills/build-loop/references/phase-6-learn.md`.

## Lazy psycopg import (v0.30.0, peer 8750d2a)

- `db`/`memory` modules now import `psycopg` lazily. The full pytest suite collects on an install that does not have the `.[db]` extra — Postgres-touching tests still skip at execution time via their own markers.
- Unblocks the pytest-collection gate below: a Python-bearing repo without the optional db extra no longer fails collection for unrelated reasons.

## Pytest-collection gate at Review-B (v0.30.1)

- `scripts/pytest_collect_gate.py` runs `pytest --collect-only` on the test surface under `env -u PYTHONPATH` semantics on Python-bearing repos at Review-B, after code-based graders and before LLM judges.
- The bar is collection only — it does NOT require the full suite to execute green (db/live tests legitimately skip at execution time via their markers); it only verifies every test module *loads*.
- A run can't report success while any test module fails to import. `pass` proceeds; `fail` routes to Iterate with `findings[]` as the rubric (each finding carries `{file, line, error_class, message}`); `skipped` (no `pyproject.toml`/`pytest.ini`/`setup.cfg` and no test paths) records `pytest_collect: skipped (<reason>)` in Review-F and proceeds.
- Non-standard layouts: when `pyproject.toml` is present but the default `scripts/`/`tests/` paths are absent, the gate skips with a loud reason — pass `--paths <dir> [...]` so a Python-bearing repo with tests elsewhere is not silently bypassed.
- Closes the silent-coverage-loss gap exposed by 8750d2a's psycopg breakage: the historical change-area test scope let a broken import quietly remove whole test modules from coverage without failing the build.

Source: `scripts/pytest_collect_gate.py`, `skills/build-loop/references/phase-4-review.md`.
