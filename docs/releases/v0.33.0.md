# v0.33.0 — Standing-roles trio: Advisor + agent-activity ledger + stakes-gated dispatch

Adds the Orchestrator / Advisor / Judge standing-roles mechanism from the 2026-06-10 model-tiering
investigation. Non-breaking: the dispatch ladder's fallback rung equals prior behavior, so a run
that does not trip a stakes trigger behaves exactly as 0.32.0 did.

## What is new
- **Agent-activity ledger** (`scripts/agent_ledger.py`): append-only JSONL, fail-open, single-writer.
  Self-certifies which model ran which role per run (closes the "could not see which tier fired" gap).
- **Advisor agent** (`agents/advisor.md`, Fable tier): owns Phase 2 plan synthesis when stakes warrant,
  dispatched through a 4-rung ladder that mirrors the existing GAP-1 auditor ladder. Rung 3 (inline,
  current model) equals today's behavior, so there is no regression floor.
- **plan-critic stakes-gating**: the orchestrator treats plan-critic WARNs as blocking only when the
  same stakes triggers fire (synthesisDensity > 5, riskSurfaceChange, stakes >= medium, or an explicit
  `dispatch_tier: frontier`); advisory otherwise.
- **`dispatch_tier: frontier`** added to the plan schema enum (resolves to Fable via `model_overrides`).
- **Doc reconciliation**: the prior claim that Fable runs Phase 1 and Phase 2 now describes the actual
  stakes-gated ladder, so docs match behavior.

## Notes
- Routing parameters are intentionally conservative. The model-tiering evidence behind the routing
  rule is directional (mostly N=1); the full experiments, equations, and the where-Fable-pays
  recommendation live in build-loop-memory/experiments.
